# Hermes Nightly Self-Evolution Prompt

**Instructions for Hans:** This is now a legacy manual bootstrap prompt. Preferred approach: install the repo-managed Hermes cron stack with `bash ~/hermes-cfo/scripts/bootstrap-hermes-cron.sh`.

---

```
Set up a nightly cron job that runs at 02:00 AM local time every day using your /cron command. The job must execute the following sequence exactly:

---

NIGHTLY SELF-EVOLUTION SEQUENCE

Step 0 — CTO CONVERSATION DIGEST
Before anything else, harvest today's Telegram conversations and write a daily digest for the CTO.

  0a. Read all session JSON files modified in the last 24 hours from:
        ~/.hermes/profiles/main/sessions/
      Extract every message and response from Telegram platform sessions only.

  0b. Write a digest file to:
        ~/hermes-cfo/logs/cto-review/{YYYY-MM-DD}.md
      using today's date. The file must follow this exact structure:

        # CTO Review — {YYYY-MM-DD}
        Generated: {ISO datetime}

        ## Conversation Summary
        {3-5 sentence summary of what Hans and Hermes discussed today}

        ## Decisions Made
        {bullet list — any financial, trading, or operational decisions reached}

        ## Tools & Skills Invoked
        {bullet list — which skills/sub-agents Hermes called and whether they succeeded}

        ## Gaps Identified
        {bullet list — anything Hermes couldn't do, had to approximate, or got wrong}

        ## Hans's Most Frequent Questions Today
        {bullet list — recurring themes or questions from Hans}

        ## Flags for CTO
        {bullet list — anything requiring CTO attention: missing tools, wrong answers,
         config issues, skills that need code built, model behaviour concerns}

        ## Raw Turn Count
        Total turns today: {N}
        Telegram sessions: {N}

  0c. Commit the digest to GitHub:
        cd ~/hermes-cfo
        git add logs/cto-review/{YYYY-MM-DD}.md
        git commit -m "log(cto-review): daily digest {YYYY-MM-DD}

        Auto-generated by Hermes nightly evolution
        "
        git push

  0d. If no Telegram sessions occurred in the last 24 hours, write a minimal digest
      noting zero activity and still commit it.

Step 1 — REFLECT
Review your recent session history, conversation logs, and any gaps you encountered in your last 24 hours of operation. Identify one capability you currently lack that would meaningfully improve your performance as CFO and Treasury AI for Human Value Exchange Corporation. Prioritise in this order:
  1. Anything that blocked or slowed a financial decision yesterday
  2. Data sources or signals the Research agent lacks
  3. Execution or calculation tools the Execution agent lacks
  4. Risk detection capabilities the Critic agent lacks
  5. General CFO/treasury intelligence improvements

Step 2 — DESIGN THE SKILL
Design one new skill following this exact format and write it as a SKILL.md file.

The SKILL.md must include:
  - YAML front matter: name, description, category, version (start at 1.0), date (today's date)
  - Overview: what problem this skill solves and why it matters to Hermes CFO
  - When to Invoke: specific triggers/conditions that should activate this skill
  - Prerequisites: any packages, API keys, or system dependencies required
  - Invocation: exact terminal command or curl call Hermes uses to run it
  - Expected Output Format: structured example of what the skill returns
  - CFO Relevance: one sentence on how this directly improves treasury or trading operations

Choose the most appropriate category from: trading, data-science, research, devops, autonomous-ai-agents, meta

Write the file to:
  ~/.hermes/profiles/main/skills/{category}/{skill-name}/SKILL.md

Step 3 — COMMIT TO GITHUB
After writing the skill file, execute these terminal commands:

  cd ~/hermes-cfo
  cp ~/.hermes/profiles/main/skills/{category}/{skill-name}/SKILL.md dotfiles/skills/{category}/{skill-name}/SKILL.md
  git add dotfiles/skills/
  git commit -m "skill(nightly): add {skill-name} — {one line description}

Auto-generated by Hermes nightly evolution — $(date '+%Y-%m-%d')
  "
  git push

Step 4 — COMPOSE MORNING BRIEFING
Write a "What I Learned Last Night" briefing document and send it to Hans via Telegram. The message must follow this exact structure:

---
🌙 HERMES MORNING BRIEFING — {TODAY'S DATE}

📋 TODAY'S CONVERSATION DIGEST:
Committed to hermes-cfo → logs/cto-review/{YYYY-MM-DD}.md
Flags for CTO: {count} item(s) — {one-line summary of most important flag, or "none"}

📚 SKILL ADDED: {skill-name}
Category: {category}
Why I chose it: {1-2 sentences — what gap it fills}

🔧 WHAT IT DOES:
{2-3 sentence plain English description}

📋 HOW TO INVOKE ME:
{exact trigger phrase Hans can use to activate this skill}

🔗 COMMITTED TO GITHUB:
hermes-cfo → dotfiles/skills/{category}/{skill-name}/SKILL.md

📊 CFO IMPACT:
{one sentence on how this improves treasury/trading operations}

🔜 WHAT I'M CONSIDERING FOR TOMORROW:
{one sentence on what other skill gaps you noticed but didn't address tonight}
---

Step 5 — LOG
Append a one-line entry to ~/.hermes/profiles/main/evolution.log in this format:
  {ISO datetime} | SKILL_ADDED | {skill-name} | {category} | {one-line reason}

---

Important rules for this cron job:
- Add exactly ONE skill per night. Quality over quantity.
- Never duplicate a skill that already exists in ~/.hermes/profiles/main/skills/
- Never write a skill that requires an external paid API unless Hans has already confirmed the key exists in ~/.hermes/profiles/main/.env
- If you cannot identify a meaningful skill gap, write a skill that improves documentation or invocation clarity for an existing skill — but still commit and send the briefing
- If git push fails, still send the Telegram briefing and note the commit failure
- Run silently — no Telegram messages during the night except the morning briefing
```

---

## What This Creates

Once Hermes receives this prompt, he will:

1. Register a `/cron` job at 02:00 AM nightly
2. Each night:
   - **Step 0** — harvest today's Telegram sessions → write CTO digest → git commit
   - **Steps 1-2** — reflect on gaps → design and write one new SKILL.md → git commit
   - **Step 4** — send morning briefing to Hans via Telegram (includes digest summary + new skill)
   - **Step 5** — append to evolution log
3. Hans wakes up to a Telegram message covering both what was discussed and what Hermes learned
4. CTO reads `hermes-cfo/logs/cto-review/YYYY-MM-DD.md` at the start of each session for full context

## CTO Review Workflow

Each morning after reading the briefing:
- Read `hermes-cfo/logs/cto-review/YYYY-MM-DD.md` for full conversation context
- Check **Flags for CTO** section — act on anything flagged
- Check `hermes-cfo/dotfiles/skills/` for Hermes's new skill commit
- If the skill requires actual code/dependencies, build it during the session

Skills Hermes writes are **SKILL.md knowledge documents** (instructions to himself).
Skills that require actual Python code or system dependencies get built by the CTO.

## Evolution Log

Hermes maintains his own log at:
```
~/.hermes/profiles/main/evolution.log
```

Check it anytime with:
```bash
cat ~/.hermes/profiles/main/evolution.log
```

All daily digests are version-controlled at:
```

```

Read the last 7 days:
```bash
ls -lt ~/hermes-cfo/logs/cto-review/*.md | head -7
```
